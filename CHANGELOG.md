# 📝 Что изменилось в ENOT SPACE

## ✅ Реализованные функции

### 1. 🔄 Синхронизация между устройствами
- Все данные автоматически синхронизируются через Firebase Firestore
- Изменения на одном устройстве мгновенно появляются на других
- Работает офлайн — данные кэшируются локально и синхронизируются при подключении

### 2. 🔐 Система авторизации
- Защищённый вход по email и паролю
- Экран входа появляется автоматически для неавторизованных пользователей
- Кнопка выхода в разделе "Ещё"

### 3. 👥 Управление пользователями (только для админа)
- Создание новых пользователей с разными ролями
- Деактивация пользователей
- Просмотр списка всех пользователей

**Роли:**
- **Админ** — полный доступ + управление пользователями
- **Помощник владельца** — полный доступ (как у админа, но без управления пользователями)
- **Менеджер** — заказы, клиенты, склад (без финансов и настроек)
- **Оператор принтера** — только заказы и принтеры

### 4. ⚡ Быстрая смена статуса заказа
- Кнопки смены статуса прямо в карточках заказов
- На дашборде: "В работу" / "Готов"
- На странице заказов: "В работу" / "Готов" / "Выдать"
- Один клик — статус изменён

### 5. 📊 Улучшенные алерты
- Уведомления о низком остатке материалов уже были
- Теперь они более заметны на дашборде
- Показываются в верхней части с иконками и цветами

### 6. 📱 Индикатор синхронизации
- В шапке приложения показывается имя пользователя
- Индикатор "● синхр..." когда данные сохраняются
- Индикатор "● ошибка" если что-то пошло не так

## 🚀 Как начать использовать

### Вариант 1: Без Firebase (как раньше)
Если не настроишь Firebase, приложение будет работать как раньше:
- Данные хранятся в localStorage
- Нет синхронизации между устройствами
- Нет авторизации (доступ у всех)

### Вариант 2: С Firebase (рекомендуется)
1. Следуй инструкции в [FIREBASE_SETUP.md](./FIREBASE_SETUP.md)
2. Настрой Firebase проект (10-15 минут)
3. Создай первого пользователя-админа
4. Войди в систему
5. Создай аккаунты для сотрудников

## 📖 Как использовать новые функции

### Создание пользователей
1. Войди как админ
2. Перейди в **Ещё** → **Управление пользователями**
3. Нажми **"+ Добавить"**
4. Заполни форму:
   - Email сотрудника
   - Имя (опционально)
   - Пароль (минимум 6 символов)
   - Роль
5. Нажми **"Создать"**
6. Передай email и пароль сотруднику

### Быстрая смена статуса заказа
1. На дашборде или в списке заказов найди нужный заказ
2. Нажми на цветную кнопку справа:
   - **"В работу"** — переводит из "Новый" в "В работе"
   - **"Готов"** — переводит из "В работе" в "Готово"
   - **"Выдать"** — переводит из "Готово" в "Выдано"
3. Статус изменится мгновенно

### Выход из системы
1. Перейди в **Ещё**
2. Нажми **"🚪 Выйти"**
3. Подтверди выход

## 🔧 Технические детали

### Структура Firebase
```
organizations/
  └── default_org/          # Данные твоей организации
      ├── orders[]
      ├── clients[]
      ├── materials[]
      └── ...

users/
  └── {userId}/             # Данные пользователя
      ├── email
      ├── role
      ├── displayName
      └── active
```

### Офлайн режим
- Данные кэшируются в localStorage
- Приложение работает без интернета
- При подключении данные автоматически синхронизируются

### Безопасность
- Все данные защищены авторизацией
- Правила Firestore ограничивают доступ
- Пароли хранятся в зашифрованном виде в Firebase Authentication

## 🐛 Возможные проблемы

### "Ошибка синхронизации"
- Проверь интернет-соединение
- Проверь правила Firestore
- Проверь конфигурацию в `src/firebase/config.js`

### "Не могу войти"
- Проверь email и пароль
- Убедись, что пользователь создан в Firebase Authentication
- Убедись, что есть запись в Firestore `users/{userId}`

### "Не вижу управление пользователями"
- Эта функция доступна только админам
- Проверь роль пользователя в Firestore

## 📞 Поддержка

Если что-то не работает:
1. Проверь консоль браузера (F12) на ошибки
2. Проверь Firebase Console → Authentication и Firestore
3. Убедись, что все зависимости установлены: `npm install`
